# MVP 產品規格

## 產品目標

讓家庭、伴侶、室友與旅行群組能在同一個空間記錄共同收支，清楚知道款項來源、使用帳號、分類、分攤方式與最終結算結果。

## 核心使用流程

1. 使用者建立群組。
2. 從聯絡人選擇成員或分享邀請連結。
3. 在群組內建立現金、銀行或其他記帳帳號。
4. 建立階層式分類。
5. 新增收入、支出或帳號間轉帳。
6. 支出可由多位付款人支付，並按平均、比例或指定金額分攤。
7. 系統彙整群組債務並提出最少筆數的結算建議。

## MVP 驗收條件

### 群組與成員

- 群組建立者自動成為擁有者。
- 支援擁有者、管理員、成員與唯讀四種角色。
- 邀請失效或被撤銷後不可加入。
- 成員離開後，既有交易仍保留顯示名稱快照。

### 帳號與交易

- 群組可建立多個帳號。
- 交易類型包含收入、支出與轉帳。
- 金額使用十進位型別，不以浮點數保存。
- 每筆修改都建立不可變的稽核紀錄。

### 分類

- 分類可包含父分類，資料層不限制深度。
- 介面預設最多展開三層。
- 已被交易使用的分類只能停用，不能直接刪除。

### 同步

- 無網路時仍可新增及瀏覽本機資料。
- 恢復網路後由 CloudKit 同步。
- 衝突採欄位級最後寫入優先，並在稽核紀錄保存修改者與時間。

## 第二階段

OCR、預算、多幣別、Widget、Shortcuts、進階洞察及自動分類。

## 已完成的垂直切片

- 建立群組表單與輸入驗證。
- 透過系統聯絡人選擇器挑選多位待邀請成員。
- 建立擁有者、待邀請成員與稽核事件。
- 使用 CloudKit Share 系統介面寄送私人、可讀寫的群組邀請。
- App 收到 CloudKit share metadata 後，將共享資料匯入 shared persistent store。
